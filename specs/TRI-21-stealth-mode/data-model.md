# Data Model: Stealth Mode

**Branch**: `TRI-21-stealth-mode` | **Date**: 2026-03-31

## Entities

### Stealth Configuration

Lives in `tricycle.config.yml`:

```yaml
stealth:
  enabled: true                    # Activate stealth mode (default: false)
  ignore_target: exclude           # "exclude" (.git/info/exclude) or "gitignore" (.gitignore)
```

**Fields**:
- `stealth.enabled` — boolean, default `false`. When `true`, all tricycle-managed paths are excluded from version control.
- `stealth.ignore_target` — string enum: `"exclude"` (default) or `"gitignore"`. Determines which file receives the ignore rules.

**State transitions**:
- `enabled: false → true` + generate → stealth rules written to target file
- `enabled: true → false` + generate → stealth rules removed from target file
- `ignore_target` changed + generate → rules removed from old target, written to new target

### Stealth Ignore Block

A demarcated block of git ignore patterns managed by tricycle:

```
# >>> tricycle stealth (do not edit this block)
.claude/
.trc/
specs/
tricycle.config.yml
.tricycle.lock
.mcp.json
# <<< tricycle stealth
```

**Target files** (mutually exclusive per config):
- `.git/info/exclude` — default, truly local
- `.gitignore` — optional, user-chosen

**Lifecycle**:
- Created when `stealth.enabled` transitions to `true`
- Removed when `stealth.enabled` transitions to `false`
- Idempotent: re-running generate with same state does not duplicate the block

### Normal-Mode Gitignore Block

The existing block generated by `cmd_generate_gitignore()`:

```
# Claude Code — keep settings.json, commands, hooks, skills committed
.claude/*
!.claude/settings.json
!.claude/commands/
!.claude/hooks/
!.claude/skills/

# Tricycle Pro lock
.tricycle.lock
```

**Relationship to stealth**: When stealth mode is active, this block is NOT written to `.gitignore`. The stealth block replaces it entirely (in the stealth target file). When stealth is disabled, this block is restored in `.gitignore` as usual.
