# Data Model: SessionStart Context Injection

## Entities

### Session Context Configuration

Stored in `tricycle.config.yml` under the `context.session_start` key.

| Field          | Type     | Default | Description                                      |
| -------------- | -------- | ------- | ------------------------------------------------ |
| constitution   | boolean  | true    | Whether to inject the constitution file           |
| files          | string[] | []      | Additional file paths to inject (repo-relative)   |

**Validation rules**:
- `constitution` must be a boolean. If absent, defaults to `true`.
- `files` entries must be strings. Non-existent files are silently skipped at runtime.
- File paths are relative to the project root.

### Context File List (.session-context.conf)

Generated artifact. One file path per line. Lines starting with `#` are comments.

```
# Auto-generated by tricycle generate settings
.trc/memory/constitution.md
docs/architecture.md
```

**Validation rules**:
- Regenerated on every `tricycle generate settings` run.
- Empty lines and comment lines are ignored by the hook.
- Paths are repo-relative (resolved against `git rev-parse --show-toplevel` at runtime).

### Hook Response

Single-line JSON output on stdout.

| Field                                         | Type   | Description                          |
| --------------------------------------------- | ------ | ------------------------------------ |
| hookSpecificOutput.hookEventName              | string | Always `"SessionStart"`              |
| hookSpecificOutput.additionalContext          | string | Concatenated, escaped file contents  |

**State transitions**:
- No files valid → no output, exit 0
- Files valid → JSON output, exit 0
- Script error → exit non-zero (Claude Code reports the error)

## Relationships

```
tricycle.config.yml
  ├── context.session_start.constitution ──→ constitution.root path
  └── context.session_start.files[] ──→ additional file paths
        │
        ▼
  .session-context.conf (generated, resolved paths)
        │
        ▼
  session-context.sh (reads conf, loads files, outputs JSON)
        │
        ▼
  .claude/settings.json SessionStart hook entry (references the script)
```
