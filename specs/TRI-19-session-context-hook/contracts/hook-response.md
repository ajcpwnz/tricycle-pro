# Contract: SessionStart Hook Response

## Protocol

The hook script receives a JSON object on stdin (session event metadata) and outputs a JSON response on stdout.

## Input (stdin)

```json
{}
```

The SessionStart hook receives minimal input. The script does not depend on stdin content.

## Output (stdout)

### Success — files injected

```json
{"hookSpecificOutput":{"hookEventName":"SessionStart","additionalContext":"## Constitution (.trc/memory/constitution.md)\n\n[file content]\n\n---\n\n## Architecture (docs/architecture.md)\n\n[file content]"}}
```

### Success — no valid files (no-op)

No output. Exit code 0.

### Error

No output. Exit code non-zero. Claude Code surfaces the error to the user.

## Content Format

Each file is injected with a header:

```
## <Derived Label> (<file path>)

<file content>

---

```

- Label is derived from the filename: `constitution.md` → `Constitution`, `architecture.md` → `Architecture`
- Files are separated by `---` horizontal rules
- Placeholder constitution (matching `Run.*trc\.constitution`) is skipped
- Empty files (0 bytes) are skipped
- Total content is truncated at 50,000 characters with a notice appended

## Config File Format (.session-context.conf)

```
# Auto-generated by tricycle generate settings
# Do not edit — regenerated on: tricycle generate settings
.trc/memory/constitution.md
docs/architecture.md
```

- One path per line, repo-relative
- Lines starting with `#` are comments (skipped)
- Empty lines are skipped
- File lives at `.claude/hooks/.session-context.conf`
